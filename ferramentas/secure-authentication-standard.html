Gestão de Sessão
Parâmetros de expiração por tipo de acesso

Sessões sem expiração configurada são não conformidade direta com 8.5. Os parâmetros abaixo são os valores máximos recomendados.

ACESSO PADRÃO
8h
Expiração de sessão após inatividade para usuários corporativos
ACESSO REMOTO
4h
Sessões remotas com re-autenticação obrigatória após este período
CONTA PRIVILEGIADA
2h
Sessões administrativas com expiração reduzida — JIT preferível
TOKEN DE API
1h
Access tokens de curta duração com refresh token controlado
INATIVIDADE
15min
Bloqueio de tela e encerramento de sessão após inatividade
RE-AUTENTICAÇÃO
Obrigatória
Elevação de privilégio ou acesso a dados restritos exige nova autenticação
⚠ Tokens permanentes: Refresh tokens sem expiração configurada ou service accounts com tokens de longa duração sem rotação são achados recorrentes em auditorias de 8.5. Revisar e configurar expiração em todos os sistemas é pré-requisito para conformidade.
Single Sign-On
Centralização da autenticação

SSO reduz a superfície de ataque ao eliminar credenciais fracas distribuídas e concentra os controles de autenticação em um ponto auditável.

RequisitoDescriçãoStatus esperado
Identity Provider únicoTodos os sistemas corporativos autenticam via IdP centralizado (Entra ID, Okta, etc.)OBRIGATÓRIO
MFA aplicado no IdPMFA configurado centralmente — não dependente de cada aplicação individualOBRIGATÓRIO
Protocolo modernoSAML 2.0, OIDC ou OAuth 2.0 — sem autenticações legadas não federadasOBRIGATÓRIO
Cobertura de sistemas críticos100% dos sistemas críticos integrados ao IdP — sem autenticação local paralelaOBRIGATÓRIO
Sistemas legadosSem integração SSO nativa — proxy de autenticação ou controle compensatório documentadoEXCEÇÃO DOCUMENTADA
Logs centralizadosSign-in logs de todos os sistemas consolidados no SIEM ou repositório de logsRECOMENDADO
Conditional AccessPolíticas de acesso condicional baseadas em risco, dispositivo e localizaçãoNÍVEL 4+

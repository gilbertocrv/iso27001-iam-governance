Entra User Script Generator

Exercício prático para traduzir um fluxo controlado de provisionamento em script PowerShell revisável. O objetivo aqui não é entregar automação de produção, mas documentar como um controle de acesso pode ser padronizado, revisado e evidenciado no contexto de IAM e ISO/IEC 27001:2022.

Provisionamento padronizado
CSV validado → script revisável → evidência operacional

A página valida o cabeçalho do CSV, normaliza os campos, gera um script com comentários de revisão e aponta linhas inválidas antes da execução. O script usa New-MgUser com PasswordProfile e ForceChangePasswordNextSignIn = $true, alinhado ao comportamento documentado para criação de usuários no Microsoft Graph PowerShell. citeturn0search0turn0search7

Como conectar isso ao projeto

Não é o produto central
Este artefato funciona como exemplo de materialização de controle, não como proposta comercial nem solução fechada.
Relacionamento com 5.18
A utilidade aqui é mostrar padronização do provisionamento, redução de erro manual e geração de evidência revisável.
Valor para governança
Você demonstra entendimento da ponte entre controle, processo, execução e auditoria — mesmo em um exercício educacional.

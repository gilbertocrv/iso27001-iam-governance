Gerador de Script PowerShell para Provisionamento

Converte um CSV padronizado em comandos New-MgUser para uso controlado no Microsoft Entra ID. A lógica roda no navegador, sem envio dos dados para backend, preservando privacidade e rastreabilidade operacional.

5.16 · Identity Management 5.18 · Access Rights A.9.2 · Provisioning Local Processing
GERADOR
Entrada CSV → Saída .ps1
Formato esperado: Nome,UPN,Cargo,Departamento. A primeira linha deve conter o cabeçalho.
O script gerado usa senha temporária ilustrativa e força troca no primeiro logon com ForceChangePasswordNextSignIn=$true. Em ambiente real, ajuste padrão de senha, fluxo de entrega segura e escopos mínimos antes da execução.
CONTEXTO DE COMPLIANCE
Como posicionar este artefato
O objetivo não é substituir aprovação humana. O artefato reduz erro operacional, padroniza entrada de dados e melhora a evidência de execução.
Risco mitigado Erro de digitação no provisionamento, inconsistência entre requisição e execução e concessão manual sem padrão mínimo de validação.
Pré-condição de controle Aprovação formal do RH e do gestor. O gerador não decide acesso; ele operacionaliza dados aprovados.
Evidência auditável Template CSV utilizado, script gerado, registro de revisão e evidência de execução no PowerShell / Entra / ticket.
Próxima evolução Validação de campos obrigatórios, catálogo de atributos, grupos por função, integração com revisão de acesso e monitoramento de novas roles.
Fluxo sugerido: aprovação → preenchimento do template → geração local do script → revisão técnica → execução controlada → arquivamento da evidência.
Limite do artefato: este gerador não valida política de segregação de funções, não verifica licença, não consulta catálogo de grupos e não deve ser executado sem revisão prévia.
